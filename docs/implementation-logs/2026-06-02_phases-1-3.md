## Technical Decisions

| Decision | Rationale | Alternatives rejected |
|---|---|---|
| Bundle clone ‚Üí copy `.git` rather than `git init` fresh | Preserves SDD commit order (specs before scaffold) which is the portfolio evidence | Route B (reinitialise) would have required manual recreation of two commits in the correct order |
| noreply email rewrite via `git rebase --root --exec` | `git filter-repo` not installed; rebase approach achieves the same result pre-push | `git filter-repo` (preferred tool, not available); amending commits post-push (rejected ‚Äî would require force push) |
| `Cast.where` actor setup moved to Cucumber `Before` hook | `Cast.where` is typed as `(actor) => Actor` (synchronous); browser launch is async | Keeping actor setup in `serenity.config.ts` (impossible without async Cast support) |
| `includes(expectedAmount)` for money assertions | Luma displays `$45.00`; spec states `"45.00"`; `includes` avoids currency symbol fragility | `equals` (breaks if currency symbol changes); regex match (overkill) |
| Serenity/JS v3 `createSerenityBDDReporter()` factory | `new SerenityBDDReporter()` requires 1‚Äì2 constructor args per TypeScript definition; factory accepts zero args | Constructor with explicit FileSystem arg (more verbose, less idiomatic) |

## Lessons Learned

- `Cast.where` in Serenity/JS v3 is synchronous-only. Any async ability setup (browser launch, HTTP client init) belongs in a Cucumber `Before` hook, not in `configure()`. The official docs show this pattern but it is easy to miss.
- `@serenity-js/serenity-bdd` v3 exports a default factory function `create()`, not a plain constructable class. `new SerenityBDDReporter()` fails without constructor args; `createSerenityBDDReporter()` works cleanly.
- `BrowseTheWebWithPlaywright.using()` takes a `Browser` (already launched instance), not a `BrowserType` (`chromium`). Inspecting the `.d.ts` file in `node_modules` resolved this quickly.
- Running `node -e "const p = require('@serenity-js/web'); console.log(Object.keys(p).join(', '))"` is the fastest way to confirm actual package exports when TypeScript errors are ambiguous.
- The CRLF `.gitattributes` issue was a NTFS/LF artefact from the bundle clone ‚Äî not a content problem. Confirmed by `git diff --stat` showing warnings but no stat output.
